### 2.3 文档格式规范

#### 1. Frontmatter（必需）

每个Markdown文件必须包含YAML frontmatter：

```markdown
---
title: Camera
sidebar_label: 相机操作
description: "相机控制相关API"
---
```

**字段说明：**
- `title`：文档标题（必填）
- `sidebar_label`：侧边栏显示名称（必填）
- `description`：文档描述（推荐）

#### 2. 文档结构模板

```markdown
---
title: [API名称]
sidebar_label: [中文名称]
description: "[功能描述]"
---

# [API名称]

[功能简介]

## 方法

### methodName

[方法描述]

**参数：**

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| param1 | string | 是 | - | 参数说明 |
| param2 | number | 否 | 0 | 参数说明 |

**返回值：**
- `Type` - 返回值说明

**示例：**
```javascript
// 示例代码
const result = fdapi.namespace.method(param1, param2);
```

## 属性

### propertyName

[属性说明]

- **类型**：`Type`
- **只读**：是/否
- **说明**：属性详细说明

## 事件

### eventName

[事件说明]

**事件数据：**
| 字段 | 类型 | 说明 |
|------|------|------|
| field1 | string | 字段说明 |

**示例：**
```javascript
fdapi.on('eventName', (event) => {
  console.log(event.field1);
});
```
```

## 七、简历亮点总结

```
项目名称：DTS Cloud 数字孪生平台 API 文档系统
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

技术栈：React 18 + Vite + Zustand + Tailwind CSS

核心功能：
1. 文档渲染引擎 - 自定义Markdown渲染器，支持GFM语法、代码高亮、API参数表格
2. 在线调试系统 - 浏览器端JavaScript沙箱，支持实时API调用和控制台输出
3. AI代码助手 - 多模型集成，API白名单验证，流式代码生成
4. 搜索系统 - 基于Fuse.js的本地全文搜索，支持中文分词

技术难点：
• 实现安全的代码沙箱执行环境
• 优化AI代码生成的准确性和响应速度
• 解决大型Markdown文档的渲染性能问题
• 实现本地搜索的中文分词支持
```
