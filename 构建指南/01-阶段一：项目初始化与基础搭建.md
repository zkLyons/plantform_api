### 1.2 UI设计风格要求

**参考风格**：Claude Code 网页端页面风格

#### 设计要求提示词

```
设计风格：Claude Code 网页端页面风格

1. 主题风格
   - 暗色主题为主，深色背景
   - 简洁、现代、专业的设计语言
   - 科技感、未来感

2. 配色方案
   - 主背景：深蓝黑色（#0a0e14）
   - 次背景：深灰蓝色（#1a1f2e）
   - 主文字：亮白色（#e6f0ff）
   - 次文字：灰蓝色（#8595a8）
   - 高亮色：青色（#22d3ee）
   - 辅助色：蓝色（#3b82f6）

3. 布局结构
   - 左侧：可折叠的侧边栏导航
   - 右侧：主内容区域
   - 顶部：固定的Header导航栏
   - 整体：简洁、不拥挤

4. 组件风格
   - 按钮：圆角、渐变背景、悬停效果
   - 卡片：圆角边框、悬停高亮
   - 输入框：深色背景、聚焦发光
   - 代码块：等宽字体、语法高亮

5. 字体选择
   - 正文：无衬线字体（Inter、SF Pro等）
   - 代码：等宽字体（JetBrains Mono、Fira Code）

6. 交互效果
   - 平滑的过渡动画
   - 悬停状态变化
   - 聚焦状态发光效果

7. 整体感觉
   - 像是一个专业的开发者工具
   - 类似 VS Code、GitHub、Claude Code 的风格
   - 简洁、高效、易用
```

## 三、详细步骤

### 3.1 创建项目

```bash
# 1. 创建Vite + React项目
npm create vite@latest my-dts-docs -- --template react

# 2. 进入项目目录
cd my-dts-docs

# 3. 安装依赖
npm install

# 4. 启动开发服务器
npm run dev
```

### 3.2 安装核心依赖

```bash
# 路由
npm install react-router-dom

# Markdown渲染
npm install react-markdown remark-gfm rehype-highlight

# 状态管理
npm install zustand

# 搜索
npm install fuse.js

# 代码高亮
npm install highlight.js

# 样式
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

### 3.3 配置项目结构

```bash
# 创建目录结构
mkdir -p src/components/Layout
mkdir -p src/components/Markdown
mkdir -p src/pages
mkdir -p src/data/docs/tutorials
mkdir -p src/data/docs/api
mkdir -p src/hooks
mkdir -p src/stores
mkdir -p src/utils
mkdir -p src/styles
```

### 3.4 配置 CSS

按照 Claude Code 风格配置 CSS 和全局样式。

## 四、组件设计要求

### 4.1 路由配置

配置基础路由：首页、文档页、调试台页面。

### 4.2 Layout组件

**设计要求**：

- 采用左侧边栏 + 右侧内容区的布局
- Header固定在顶部
- 整体风格简洁、现代

### 4.3 Header组件

**设计要求**：

- 左侧：Logo + 品牌名称
- 中间：导航链接（首页、文档、在线调试）
- 右侧：搜索按钮、主题切换按钮、移动端菜单按钮
- 固定在顶部，不随内容滚动

### 4.4 Sidebar组件

**设计要求**：

- 左侧固定，可折叠
- 显示文档分类（开发教程、API文档）
- 分类可展开/折叠
- 当前文档高亮显示
- 支持滚动

### 4.5 Footer组件

**设计要求**：

- 显示品牌信息
- 快速链接（快速开始、API文档、在线调试）
- 外部资源链接（GitHub等）
- 版权信息

### 4.6 首页

**设计要求**：

- Hero区域：标题、描述、CTA按钮
- 功能卡片：API文档、在线调试、AI助手、图层管理
- 快速链接：常用文档入口
- 统计数据：API类数量、方法数量、文档数量

### 4.7 文档页面

**设计要求**：

- 显示文档标题
- 渲染Markdown内容
- 代码块有语法高亮
- 表格正确显示
- 支持GFM语法

## 六、常见问题

### Q1: 项目启动失败

A: 检查Node.js版本（需要18+），检查依赖是否安装完整

### Q2: 路由不生效

A: 检查React Router版本，确保使用v6

### Q3: 样式不生效

A: 检查Tailwind CSS配置，确保content路径正确

### Q4: Markdown不渲染

A: 检查react-markdown版本，确保插件正确安装
