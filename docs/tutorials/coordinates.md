### 坐标系类型

DTS Cloud 支持两种主坐标系：

| 类型 | 说明 | 典型用途 |
|------|------|----------|
| **PCS**（投影坐标系） | 本地平面直角坐标，单位：米 | 工厂、园区、建筑等局部场景 |
| **GCS**（地理坐标系） | 经纬度坐标（WGS84 / CGCS2000） | 城市级、全球级场景 |

当前场景使用哪种坐标系，由服务端配置决定，可通过 `fdapi.coord` 查询。

### 坐标数组格式说明

SDK 中多边形、折线等几何体使用多维数组表示坐标。`z`（高度）可省略，默认为 0。

#### 单个点（一维数组）

```js
let point = [x, y, z]; // z 可省略
```

#### 单条折线 / 单个环（二维数组）

坐标点需按**顺时针或逆时针**顺序排列：

```js
let z = 0;
let ring = [
  [1, 1, z],
  [2, 2, z],
  [3, 3, z],
];
```

#### 单个 Part（含内环的多边形，三维数组）

```js
let z = 0;

// 1 个 Part，无内环
let c1 = [
  [1, 1, z],
  [2, 2, z],
  [3, 3, z],
];

// 1 个 Part，含 1 到多个内环
// 第一个元素是外环，其余是内环（孔洞）
let c2 = [
  [[1, 1, z], [2, 2, z], [3, 3, z]],  // 外环
  [[1, 1, z], [2, 2, z], [3, 3, z]],  // 内环 1
  [[1, 1, z], [2, 2, z], [3, 3, z]],  // 内环 2
];
```

#### 多个 Part（四维数组）

```js
let z = 0;

// 多个 Part，有的含内环，有的不含
let c3 = [
  // 无内环的 Part
  [
    [[1, 1, z], [2, 2, z], [3, 3, z]],
  ],
  // 含内环的 Part（第一个元素是外环，其余是内环）
  [
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 外环
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 内环
  ],
];

// 多个 Part，每个 Part 都含内环
let c4 = [
  [  // Part 1
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 外环
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 内环
  ],
  [  // Part 2
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 外环
    [[1, 1, z], [2, 2, z], [3, 3, z]],  // 内环
  ],
];
```

### 坐标转换

使用 `fdapi.coord` 在各坐标系之间转换：

```js
// PCS ↔ GCS 互转
const gcs = fdapi.coord.pcsToGcs([488000, 2890000, 10]);   // → [lng, lat, alt]
const pcs = fdapi.coord.gcsToPcs([120.15, 30.27, 10]);     // → [x, y, z]

// 屏幕坐标 → 世界坐标
const world = fdapi.coord.screenToWorld([960, 540]);

// 世界坐标 → 屏幕坐标
const screen = fdapi.coord.worldToScreen([488000, 2890000, 10]);
```

详细参数说明见 [Coord API 文档](/docs/api/utils/coord)。
