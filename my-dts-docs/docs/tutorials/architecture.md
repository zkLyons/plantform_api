# 架构概览

第一次接触 DTS Cloud SDK，建议先花 10 分钟读完本文，建立整体心智模型，再去看具体 API。理解了下面这几件事，后续 100 多个对象的用法都会变得自然：场景在哪里渲染、用什么对象连接、为什么接口是异步的、场景分哪两类、坐标怎么算、对象怎么增删改。

## 一、它是什么：云端渲染 + 视频流

DTS Cloud 是**云渲染**数字孪生平台——三维场景并不在浏览器里渲染，而是运行在云端的高渲染服务（ACP / AirCloud 云渲染）上：

- 云端在 GPU 上实时渲染三维场景（3DT 工程），把画面编码成**视频流**（像素流 / WebRTC 方式）推送到网页，网页只负责显示这段视频并采集交互。
- 你的 JavaScript 通过 SDK 把**指令（JSON 命令）**经 WebSocket 发给云端，云端执行后改变场景，再把新画面回传。

```text
   浏览器（网页）                                云端高渲染服务
 ┌──────────────────────┐                       ┌──────────────────────┐
 │ fdplayer  视频流画面 │ ◀──── 视频流 ─────    │                      │
 │ (DigitalTwinPlayer)  │ 像素流                │ 三维场景（3DT 工程） │
 │                      │                       │ 相机 / 模型 / 图层   │
 │ fdapi   接口调用     │ ──── JSON 命令 ───▶   │ 云端 GPU 实时渲染    │
 │ (DigitalTwinAPI)     │ WebSocket / WebRTC    │                      │
 └──────────────────────┘                       └──────────────────────┘
   鼠标 / 键盘 / 触摸交互 ────────────────────▶
```

这套架构带来三个推论，贯穿全部 API：

1. **浏览器端不需要 WebGL / 高性能显卡**，移动端也能流畅接入；性能瓶颈主要在网络与云端实例。
2. **海量三维数据留在云端**，前端不下载模型，加载快、数据安全。
3. **接口天然是异步的**——每次调用都要把命令发到云端、等执行结果回传（详见第三节）。

## 二、三个全局对象

整个 SDK 的入口就三个全局对象，先认清它们的分工：

| 对象 | 由谁创建 | 职责 |
|------|----------|------|
| `fdplayer` | `new DigitalTwinPlayer(host, option)` | 云渲染**视频流窗口**：在网页指定 `domId` 容器里显示三维画面，承载键鼠/触摸交互；通过 `iid`/`pid` 等指定连接哪个实例、加载哪个工程。 |
| `fdapi` | `new DigitalTwinAPI(host, options)` | **API 总入口**：相机、标注、图层、仿真等所有功能都通过 `fdapi.xxx.方法()` 调用，是连接与调用的核心。 |
| `HostConfig` | SDK 提供的全局配置 | 全局路径/配置对象，最常用 `HostConfig.Path` 拼接云端资源路径（贴图、模型、视频等），例如 `HostConfig.Path + '/assets/image/decal.png'`。 |

两个关键注意点：

- 只有 `new DigitalTwinAPI(...)` 之后，全局 `fdapi` 才有效；并且**必须在 `onReady`（工程加载成功）回调之后再调用 API**，否则会有崩溃或错误风险。
- 一个页面创建了多个视频流窗口时，`fdapi` 始终指向**最后一次**创建的实例，多窗口场景需自行保存各自的实例引用。

## 三、为什么接口是异步的

因为每次 API 调用本质是「把命令发到云端 → 等待执行 → 拿回结果」的一次往返，所以 `fdapi` 的大部分接口都是异步的。SDK 同时支持三种写法，**同一个函数体内只能选一种，不要混用**：

- **Callback 回调**：最基础，链式调用多时嵌套深。
- **Promise `.then()`**：解决回调嵌套，可读性更好。
- **`async / await`**：写法最接近同步代码，推荐。

具体写法与示例见 [二次开发：异步调用方式](/docs/tutorials/async-call)。

## 四、两类场景：投影坐标系 vs 球面坐标系

DTS 场景分两大类，**写代码前先确认你的场景属于哪一类**，因为部分对象与坐标含义不同：

- **投影坐标系场景**（最常用）：本地平面直角坐标系，长度/角度/面积恒定，适合园区、厂区、城市局部等高精度场景。在线调试台的「**投影**」示例（约 1000+ 条）对应这一类。
- **球面（椭球）坐标系场景**：基于数字地球，用经纬度组织，适合全球或大范围 GIS。部分对象**专用于球面**，如 `GlobeTerrain`（球面地形）、`ImageryLayer2`（球面影像服务）；调试台的「**球面**」示例对应这一类。

影像、地形等少数能力在两类场景下有不同对象/实现，选 API 时留意文档中标注的适用坐标系。

## 五、坐标系与转换

云渲染支持四类坐标系，可用 `Coord` 工具互转（详见 [坐标系与坐标转换](/docs/tutorials/coordinates)）：

| 坐标系 | 含义 |
|--------|------|
| 屏幕坐标 | 以像素表示的窗口位置，左上角为原点；交互输入用得多。 |
| 场景坐标（世界坐标） | 三维空间直角坐标 [X, Y, Z]，东为 X+、北为 Y+、上为 Z+。 |
| 地理坐标（GCS） | 经度/纬度/高程，如 WGS84、CGCS2000。 |
| 投影坐标 | 二维平面投影坐标，长度/角度/面积恒定，也称本地坐标系。 |

旋转统一用**欧拉角** `[Pitch, Yaw, Roll]`（俯仰/航向/翻滚），取值范围 Pitch `[-90,90]`、Yaw `[-180,180]`。

## 六、对象生命周期与批量更新

绝大多数可视化对象（继承自 `BaseObject`）遵循统一的生命周期：

- **创建** `add(data)` → **查询/修改** `get()` / `update()` / `setXXX()` → **销毁** `delete(ids)` / `clear()`。
- 需要一次改大量属性时，用 **批量更新模式**显著提升性能（把多次修改合并成一次往返，避免逐次通信）：

```js
fdapi.xxx.updateBegin();              // 开始批量
for (let i = 0; i < 1000; i++) {
  fdapi.xxx.setColor(i, Color.Yellow);
}
fdapi.xxx.updateEnd(function () {     // 提交，异步
  log('update finished!');
});
```

注意：`updateBegin()` 是同步的、不需要 `await`；`updateEnd()` 是异步的，可用回调或 `await`。
